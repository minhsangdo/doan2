P59: 4. SƠ ĐỒ LUỒNG DỮ LIỆU (DFD)
P60: 4.1 DFD Mức 0 (Context Diagram)
P61: Hệ thống là hộp đen nhận input từ Thí sinh và Admin, tương tác với Neo4j và LLM API, xuất kết quả trả lời:
P62: Luồng vào: Câu hỏi từ thí sinh → Hệ thống chatbot
P63: Luồng vào: Dữ liệu tuyển sinh PDF → Admin → Hệ thống
P64: Luồng ra: Câu trả lời tư vấn → Thí sinh
P65: Luồng ra: Báo cáo thống kê → Admin
P66: Kho dữ liệu: Neo4j Knowledge Graph (hai chiều)
P67: Dịch vụ ngoài: OpenAI / Anthropic API (hai chiều)
P68: 4.2 DFD Mức 1 - Phân rã các tiến trình chính
P69: Hệ thống được phân rã thành 4 tiến trình chính:
P70: P1 - Xử lý câu hỏi: Nhận input → tokenize → phân loại intent → tạo query vector
P71: P2 - Graph RAG Retrieval: Vector search trên Neo4j + duyệt đồ thị quan hệ → tập hợp context
P72: P3 - Sinh câu trả lời: Gửi context + câu hỏi lên LLM API → nhận response → format
P73: P4 - Quản lý Knowledge Graph: Import PDF → extract → normalize → insert Neo4j → embed
P74: 4.3 DFD Mức 2 - Chi tiết P2 (Graph RAG Retrieval)
P75: P2.1 - Vector Similarity Search: Embedding câu hỏi → cosine similarity với db.index.vector → Top-K nodes
P76: P2.2 - Graph Traversal: Từ Top-K nodes → duyệt 1-2 hop quan hệ (BELONGS_TO, HAS_SCORE, USES_COMBO)
P77: P2.3 - Context Assembly: Gộp thông tin node + relationship + metadata thành chuỗi context
P78: P2.4 - Reranking: Đánh trọng số context dựa trên độ liên quan và độ mới của dữ liệu
P79: 4.4 DFD Mức 2 - Chi tiết P4 (Knowledge Graph Builder)
P80: P4.1 - PDF Extraction: pdfplumber/PyMuPDF trích xuất text, bảng từ file PDF
P81: P4.2 - LLM Entity Extraction: Prompt LLM để nhận diện entity (Nganh, TohopMon, DiemChuan, PhuongThuc)
P82: P4.3 - Entity Resolution: Chuẩn hóa tên ngành, dedup, gán ID chuẩn theo mã ngành
P83: P4.4 - Cypher Generation: Tạo lệnh MERGE cho node và relationship
P84: P4.5 - Batch Insert: Nhóm lệnh Cypher, thực thi theo batch qua neo4j-driver
P85: P4.6 - Embedding & Indexing: Gọi OpenAI Embeddings API → lưu vector → tạo vector index

P197: 9. CƠ SỞ DỮ LIỆU NEO4J
P198: 9.1 Thống kê dữ liệu DNC 2025
P200: 9.2 Cypher Query mẫu cho RAG
P201: // Query tra cứu điểm chuẩn ngành Y khoa
P202: MATCH (n:Nganh {ma_nganh: '7720101'})-[:HAS_SCORE]->(d:DiemChuan)
P203: RETURN n.ten, d.diem_thpt, d.diem_hocba, d.diem_dgnl, d.diem_vsat
P205: // Query lấy tất cả ngành nhóm Y-Dược kèm điểm
P206: MATCH (ng:NhomNganh {ten: 'Y-Duoc'})<-[:BELONGS_TO]-(n:Nganh)
P207: MATCH (n)-[:HAS_SCORE]->(d:DiemChuan {nam: 2025})
P208: RETURN n.ten, n.ma_nganh, d.diem_thpt ORDER BY d.diem_thpt DESC
P210: // Vector search + graph context (Graph RAG query)
P211: CALL db.index.vector.queryNodes('entity_embedding', 5, $question_embedding)
P212: YIELD node, score
P213: MATCH (node)-[:HAS_SCORE]->(d:DiemChuan)
P214: MATCH (node)-[:USES_COMBO]->(t:TohopMon)
P215: RETURN node.ten, d.diem_thpt, collect(t.ten) as tohop, score
P216: ORDER BY score DESC
